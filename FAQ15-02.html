Windowsのドメイン管理者権限では権限が不足、ローカル管理者権限が必須

E-Post Mail Server (x64) インストール時や、Windows Serverログイン時および E-Post Mail Control を起動する際に、ドメイン管理者権限では基本的に権限が不足します。特に Active Directory のメンバーサーバとなっているマシンでインストールしたり、運用するときには注意してください。ローカル管理者権限、もしくはローカル管理者権限が付与されている Administrator で行う必要があります。

たとえば、E-Post関連のサービスが稼働中であるにもかかわらず、E-Post Mail Control 画面にあるサービスのボタンが［登録］の表示になってしまう場合、その原因の一つに E-Post Mail Control（Epstcontrol.exe）がローカル管理者権限で実行されていないことが理由になることがあります。
また、インストール時に作成される「E-Post Mail Server Enterprise for JP」アイコンから E-POST Mail Control を起動するときには、明示的に右クリックメニューから「管理者として実行」を選択するようにしてください。
