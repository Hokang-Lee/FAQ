ライセンスの増加などユーザー数が変更されたライセンスキー登録を行う場合

Standard版をご使用のケースで、ライセンス登録済みの状態から、ライセンスの増加などに伴い、変更されたユーザー数用のライセンスキーを再登録する場合は、下記の方法で登録済みのライセンスキーをいったんクリアする必要があります。クリア後、新しいライセンスキーの登録を行ってください。Enterprise II 版の場合はこの記事の内容は関係ありません。

■64bit版 E-Postシリーズ製品：E-Post Mail Server Standard (x64) ／ E-Post SMTP Server Standard (x64)の場合
 次のファイルをダウンロードし解凍してください。
　●init-license.zip　…全環境
E-Post Mail Controlを閉じてから、解凍された.reg ファイルをダブルクリックし、警告メッセージに対して「はい」をクリックすることで、入力済みのライセンス情報がクリアされます。
なお、ライセンス登録情報のクリア後は、自動的に試用版状態に移行しますので十分ご注意ください。試用版時期の消化が終わっていれば、サービスは応答しなくなる状態になることがあります。
その後、再度 E-Post Mail Controlを開き、「バージョン情報」タブに切り替え、新たな「ライセンスキー」を登録してください。
なお、Enterprise II (x64) 版の場合は、年次更新ごとにライセンスキーを追加登録していただきますので、次年度更新のタイミングでは、このクリア作業を逐一行っていただく必要はありません。

その後、再度 E-Post Mail Controlを開き、「バージョン情報」タブに切り替え、新たな「ライセンスキー」を登録してください。

一方、E-Post BossCheck Optionのライセンスキー自体では、ライセンス数の管理を行っておりませんので、リセットは必須ではありませんが、リセットしたいときには次の方法で行ってください。

■64bit版 E-Post BossCheck Option (x64) の場合
 次のファイルをダウンロードし解凍してください。
　●bosscheck-reset.zip　　…　全環境
E-Post BossCheck Option (x64) の画面を閉じてから、解凍された.reg ファイルをダブルクリックし、メッセージに対して「はい」をクリックすると、入力済みのライセンス情報がリセットされます。
その後、再度 E-Post BossCheck Option (x64) の画面を開き、新たな「ライセンスキー」を登録してください。

■32bit版 E-Post BossCheck Option の場合
 次のファイルをダウンロードし解凍してください。なお、32bit版E-Postシリーズが64bitOSにインストールされているときは、下の方をダウンロードしてください。
　●bosscheck-reset.zip　　…　Windows 2003 / Windows 2008 などの32bitOS環境
　●bosscheck-reset(64).zip　…Windows 2008 R2 / Windows 2012 / Windows 2012 R2などの64bitOS環境
E-Post BossCheck Option の画面を閉じてから、解凍された.reg ファイルをダブルクリックし、メッセージに対して「はい」をクリックすると、入力済みのライセンス情報がリセットされます。
その後、再度 E-Post BossCheck Option の画面を開き、新たな「ライセンスキー」を登録してください。
