サービス停止後に再開が突然できなくなった
ＯＳのhotfixを適用している場合で、サービスのインストールフォルダが、"C:\Program Files\spa-pro\" (ドライブは任意)のようにフォルダパスに一部スペースが使われているフォルダにインストールしてある場合、サービスの開始ができなくなることがあります。以下レジストリを開き、サービス実行パスをショートネームに変更して保存すると正常に起動できるようになります。

ただし、この問題が発生した場合は、他のトラブルが発生する可能性がありますので「空白」のない別フォルダにインストールし直すようにして下さい。
[SPA-PRO Mail Server 2003/SPA-PRO Mail Server/SPA-PRO Mail(SMTP) @Solomon]の変更先
HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\SPARS-PRO\ImagePath
現在 -> C:\Program Files\SPA-Pro\spa-rs.exe
変更 -> C:\PROGRA~1\SPA-Pro\spa-rs.exe

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\SPADS-PRO\ImagePath
現在 -> C:\Program Files\SPA-Pro\spa-ds.exe
変更 -> C:\PROGRA~1\SPA-Pro\spa-ds.exe

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\SPAPOP3S-PRO\ImagePath
現在 -> C:\Program Files\SPA-Pro\spa-pop3s.exe
変更 -> C:\PROGRA~1\SPA-Pro\spa-pop3s.exe

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\SPAIMAP4S-PRO\ImagePath
現在 -> C:\Program Files\SPA-Pro\spa-imap4s.exe
変更 -> C:\PROGRA~1\SPA-Pro\spa-imap4s.exe
