Roundcube mail を E-Post と組み合わせて設定、添付ファイルを送信したときの問題 new!

Roundcube mail を E-Postと組み合わせて設定、添付ファイルを送信したとき "internal server error" のエラーが返される現象や、さらに Roundcube に添付ファイルをアップロードしたとき添付ファイル名がファイル名+[size=ファイルのバイト数]として変更されてしまう現象が報告されたことがありました。
これらの現象は最終的に解決したのですが、そのときのサポート担当・開発担当とユーザー様とのやり取りを参考までに掲載いたします。

添付ファイルを送信したとき "internal server error" のエラーが返される現象

Roundcube mail にて添付ファイルを添付すると "internal server error" のエラーが返されるという現象については、Roundcube mail や E-Postのモジュールについて以下を見直しました。
・Roundcube mailのバージョン確認→ver1.6.9、ver1.6.11
・php.iniファイルの内容
・defaults.inc.phpファイルの内容
・E-Post IMAP4サービス（EPSTIMAP4S）のバージョン確認と最新バージョン適用

---------------（報告への初回返答）---------------
(1) 添付ファイルのサイズ上限設定として、[roundcubemailフォルダ]\config 下にdefaults.inc.php という設定ファイルがあるので、このファイル内に何らかの値が設定されていないか。
(2) Roundcube mail 及び EPSTIMAP4S バージョン確認。
(3) [roundcubemailフォルダ]\temp フォルダのセキュリティ設定において IIS_IUSRS で書き込み権限を与えているかどうか。
(4) 以下は未確認ながら下記URLでのやり取りにて添付ファイルサイズの設定値について情報あり。
[添付ファイルのサイズ制限について]
（※日本語訳）
phpフォルダの php.ini または roundcoubeフォルダの .htaccessファイル内でphpパラメータ "upload_max_filesize" と "post_max_size"を変更する必要があります。「upload_max_filesize」を何に変更するにしても、「post_max_size」は少なくとも1M高くする必要があります。
　　　　　　　　　↓
これについて、php.ini ファイルの"upload_max_filesize" と "post_max_size"を変更後、IISを再起動すれば、上限変更されるとの開発担当からの報告がありました。

E-Post Mail Server (x64)の IMAP4サービス（EPSTIMAP4S）に組み合わせる形で、異なるOS環境でそれぞれ RoundCube mail のインストールと設定を行い、下記全てのOS環境で添付ファイルのアップロードが正常に行えることを確認しました。いずれも、"C:\Roundcubemail\temp"フォルダに対する「IIS_USERSを追加し、アクセス権で(読み書き）可能」の権限は与えており、さらに E-Post IMAP4サービス（EPSTIMAP4S）は ver4.71以降の最新モジュールを使いました。
Windows Server 2012R2 / 2016 / 2019 / 2022 / 2025

（※補足情報）
php.ini にある"upload_tmp_dir ="の英文説明を見たところ、以下のように記載されていました。
; Temporary directory for HTTP uploaded files (will use system default if not
; specified).
; http://php.net/upload-tmp-dir
（※日本語訳）「指定されない場合、PHPはシステムのデフォルト設定を使用します。」

「システムのデフォルト設定」がどのようになっているかを確認するため、Roundcubemail のルートフォルダに以下の内容の test.php を作成、デフォルト設定としてアサインされている一時フォルダを確認しました。
[test.php]
------------------------
<?php
echo sys_get_temp_dir()
?>
------------------------
ブラウザから "http://localhost/roundcubemail/test.php" で表示させたところ、デフォルト設定としてアサインされている一時フォルダが確認できます。
---[表示結果]-----------
C:\Windows\TEMP
---------------------------
"C:\Windows\TEMP" の該当フォルダに「IIS_USERSを追加し、アクセス権で(読み書き）可能」の権限を与えると、"upload_tmp_dir ="未定義とした場合でも、添付ファイルのアップロードが正常に行えるのを確認しました。php.ini の変更やフォルダへの権限変更があった後は、いずれの場合も、IISの再起動を行って試しました。

添付ファイルをアップロードしたとき添付ファイル名がファイル名+[size=ファイルのバイト数]として変更されてしまう現象

---------------（ユーザーの報告）---------------
phpフォルダの php.ini ファイル内の
;upload_tmp_dir =
コメントアウトされていましたので
upload_tmp_dir = c:\roundcubemail\temp
上記に編集したところ、添付ファイルの利用が可能になりました。
しかし、Roundcube にアップロードした添付ファイル名が以下のように Roundcube 上で変更されてしまいます。
（変更後の状態）ファイル名+[size=ファイルのバイト数]
（例）「test.txt」 を添付ファイルとしてアップロード送信後に Roundcube で表示されるファイル名が「test.txtsize=12」となる
ただし、Roundcubeから送信していても、受信先がGmailで閲覧しているユーザーのアカウントでは添付ファイルのファイル名は正常な表示となっています。

---------------（上記報告への返答）---------------
Roundcube にアップロードした添付ファイルの名前が Roundcube 上で変更されてしまう現象ですが、 E-Post IMAP4サービス（EPSTIMAP4S）のモジュールが最新版のときは、ファイルのバイト数がファイル名の後ろに付いてファイル名表示が変わることは確認できませんでしたが、下記のように EPSTIMAP4Sのモジュールが古いときには、現象が確認でき、最新のものに適用したところ、改善されることが確認できました。
EPSTIMAP4S. v4.67 で自分宛にメール送受信したところ、ファイル名+[size=ファイルのバイト数]と表示されました。これは、EPSTIMAP4Sの旧バージョンが抱えている問題のようです。最新版EPSTIMAP4S v4.71以降に差し替えた結果、"ファイル名"だけの表示に改善されます。
もし別のメーラーで受信すると名称がおかしくなるというケースであれば、メールクライアント側の問題になります。
