どの程度の大量配送が可能ですか？

シングルサーバの Standard 版では、理論値最大30万通／時が可能です。ただネットワーク回線の混雑やDNSサーバの負荷など、運用環境によって数値は大きく左右しますので、あくまで参考値にしてください。

アンチウイルス機能の付いた Enterprise II 版では、Standard 版に比べ約1/3〜1/10程度にパフォーマンスが下がります。
また、クラスタモデル版はフェイルオーバー＆フェイルバックの冗長化を主眼に置いています。特にE-Post方式クラスタでは、Ethernetで接続されたネットワーク共有ドライブをメール作業フォルダとして使用することから、配送能力はローカルドライブをメール作業フォルダとして使用するシングルサーバ版より、パフォーマンスはかなり落ちてしまいます。一方、MSFC方式のクラスタでは、高速なFCやiSCSIの専用ストレージを共有ディスクとして使用しますので、パフォーマンスは落ちることはないとされています。

大量配送を意図するときは、シングルサーバ版での E-Post Mail Server Standard (x64) を1台、または E-Post SMTP Server Standard (x64) を1台で送信する方法などを検討されるとよいでしょう。
